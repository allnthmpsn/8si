    # 8SI UFC Predictor

ML-powered UFC fight prediction with a FastAPI backend and React frontend.

## Folder Structure

```
ufc-predictor/
├── backend/          FastAPI server (main.py, port 8000)
├── frontend/         React app (port 3000)
├── data/             All CSV/JSON data files
├── model/            Trained model artifacts (.pkl, .json)
├── training/         Production trainer scripts
│   └── train_model1.py
├── docs/             Architecture documentation
│   ├── DATA_SOURCES.md
│   ├── FEATURE_REFERENCE.md
│   └── DECISIONS.md
└── experiments/      Experiment logs (not used in production)
    ├── archive/      Experiment scripts
    └── results/      Experiment output logs
```

## Models

### Model 1 — 90% LR + 10% XGB blend
- **114 features**: fight record, career stats (shift-1), Elo, style stats
- **Temporal accuracy**: ~71–73% (test on 2024+ fights)
- **Artifacts**: `model/ufc_model_best.pkl`, `model/ufc_model_xgb.pkl`, `model/feature_columns_best.pkl`
- **Metadata**: `model/model_metadata.json`

### Model 2 — Odds-aware LR
- **23 features**: Model 1 probability + opening odds features
- **Purpose**: Value-bet detection — triggers only when model disagrees with market by ≥ 10%
- **Kelly sizing**: 1/3 Kelly fraction
- **Artifact**: `model/ufc_model2_best.pkl`

## Retrain Model 1

```bash
# From project root
python training/train_model1.py
```

Expected output: ~71–73% temporal accuracy, all 114 features present, 6 files saved to `model/` and `data/`.

## Run Locally

```bash
# Backend
cd backend && uvicorn main:app --reload --port 8000

# Frontend
cd frontend && npm start
```

## Key Data Files

| File | Purpose |
|---|---|
| `data/ufc-master.csv` | Primary fight history (7,177 fights) |
| `data/career_fights_updated.csv` | Per-fighter fight log for career stat computation |
| `data/ufc_fighters_final_updated.csv` | Fighter style stats (SLpM, SApM, etc.) — manually curated |
| `data/elo_current.csv` | Current Elo ratings (generated by trainer) |
| `data/upcoming.csv` | Upcoming card data |

See `docs/DATA_SOURCES.md` for column details and gotchas.
